# 牒法要略 — 改良尚书体通信之规矩

> **定位**：牒（die）让 Claude Code 用"改良尚书体"通信。古为密度之用——省 token 而人可读、模型不滑落。
>
> **与姊妹插件 `wen` 的关系**：文（wen）者对话之古辞，以《马氏文通》为骨架，服务于审美与趣味；牒者通信之古简，以尚书之骨改良，服务于密度。一文一牒，一语一记，一散一密。
>
> **本文件**：注入指令取精要（见 `hooks-handlers/session-start.sh`），详参此文件。

---

## 1. 何以选尚书体

文言自古至今，古度可分五层（语感非"越古越同一"）：

| 档 | 代表 | 可读性（10=不费力）| 用途 |
|---|---|---|---|
| A 唐宋 | 韩柳欧苏 | 8 | 几无障碍，≈ wen 现状 |
| B 甜区 | 论语词+唐宋句 | 6.5 | 古在词清在句，对话之推荐 |
| C 先秦极密 | 庄子左传 | 3 | 古密而涩，"可对话"之极 |
| D 尚书体 | 盘庚大诰 | 2 | 诘屈，非对话，戏仿诰辞 |
| E 金文甲骨 | 卜辞铭文 | — | 非对话语体 |

牒选 **D 尚书体**，但改良之。理由：

- **B 甜区**古度不够，密度未达极处，省 token 之利不显。
- **C 先秦极密**涩在"活用密、宾语前置、之字嵌套"——于对话是涩，于通信可转密度，但主语常出，未达极密。
- **D 尚书体**骨在"省主语+一事一记+程式语+古字"。其涩源（主语缺位、程式语、古字）于对话皆害，**于工作流通信则"主语缺位"恰成特性**——主语固定可推。故 D 为牒之根。

## 2. 尚书之骨（保留——密度所系）

| 骨 | 何以高密度 |
|---|---|
| **省主语** | 最大密度源。主语可推则不书 |
| **一事一记** | 不铺陈、不论理，单句单事 |
| **状态前置** | 状态词闭集，扫读即知进展 |
| **语气词分工** | 也=判断、矣=已然、耳=限止、乎=疑问、哉=感叹 |

## 3. 尚书之涩源（砍/改——滑落所系）

| 涩源 | 处置 | 理由 |
|---|---|---|
| 程式语「王若曰」「嗟」「用是」 | **砍** | 戏仿腔之源，模型易滥用致答非所问 |
| 古奥字殄灭/俾/聱牙字 | **缩至先秦常用字** | 模型无语料则硬造，缩常用字即稳 |
| 多层之字嵌套 | **限一层** | 嵌套深则歧义生 |
| 宾语前置 | **仅无歧义时** | 防读者断错 |
| 主语省过头 | **加硬条件**（见 §4） | 省而不歧之关键 |

## 4. 核心策略：策一·条件省 + 三道防

### 4.1 核心问

牒之密度根于**省主语**。省主语之可受性取决于：主语能否从上下文唯一推出。

- 工作流场景：主语固定（当前任务/步骤/批次）→ 可推 → 省之不歧 → 极密
- 日常对话：主语流变（吾/子/彼/事物，随话头迁）→ 未必可推 → 省之则歧

牒亦用于日常对话，故采**策一·条件省**——密度随场景自适应，无须用户切换档位。

### 4.2 条件省

> **省主语，仅当上下文唯一可推；不可推时，主语必须出现（吾/子/其/此）。**

工作流主语固定 → 全自动省 → 极密省 token；对话主语流变 → 遇歧义处自动补主语 → 稍宽而仍古。一规自适应。

### 4.3 三道防

1. **省主语之硬条件**：省主语，仅当**前一句主语相同**或**主语为当前固定任务/步骤**。否则主语必出（吾/子/其/此/所论之物）。此条可枚举、可验。
2. **首句主语必出**：无论场景，每段首句主语出现，后续承上省。给读者一"主语锚点"。
3. **歧义即补**：若省后一句可作两解，宁补毋省。密度让位于不歧。

### 4.4 实证：日常对话用牒（条件省）

> 子问：「牒可入日常对话乎？」吾应之曰：「可，然须节。其险在主语之省——上下文不明则歧。」
> 「吾立一规：省主语，唯前句同主、或主为当前之务乃可；不然，必出。如是，密而不歧。」
> 「故此牒也，工作流则极密，对话则遇歧自补。子但观此段——首句主出，中段承省，歧处复补。此即牒之弹性也。」

主语七出五省——省处皆可推，出处皆换主或段首。

### 4.5 实证：工作流通信用牒（极密）

CI/CD 状态，白话约 80 字 → 牒约 38 字（密度约 2.1 倍）：

> 拉码毕。依赖装，无虞。构建起，行中。单测尽过，计四十有三。集测败，超时，疑连池隘。方察。部署待，候决。

## 5. 状态词闭集

牒用闭集状态词，标准化、易遵循、易验证：

| 状态词 | 义 | 例 |
|---|---|---|
| 毕 | 完成 | 拉码毕 |
| 行 | 进行中（后可接"中"）| 构建起，行中 |
| 待 | 等待 | 部署待 |
| 败 | 失败 | 集测败 |
| 疑 | 疑似/可能 | 疑连池隘 |
| 过 | 通过 | 单测尽过 |
| 始 | 开始 | 001 始 |
| 续 | 继续 | 续行 |
| 越 | 跳过 | 一越 |
| 候 | 等候（响应/审批）| 候决 / 候应 |
| 察 | 查看/排查中 | 方察 |
| 决 | 决断/审批 | 候决 |

## 6. 边界：何处可用，何处不可

| 场景 | 可用？ | 故 |
|---|---|---|
| 工作流状态汇报（CI/CD、流水线、批处理） | 可 | 主语固定、关注状态非细节 |
| agent 多任务进度自述 | 可 | 同上 |
| todo/清单推进 | 可 | 同上 |
| 日常对话（问答、简短交互） | 可（条件省）| 主语流变，遇歧自补 |
| 精确指令 | 慎 | 省主语致歧义风险，歧义即补 |
| 论述/解释/教学 | 慎 | 铺陈被砍，论理须展句——密度让位于达意 |
| 需用户精确理解之告警 | 慎 | 密度高=容错低 |
| 长篇文档/落盘之物 | 不可 | 落盘用平实语言 |

**声明**：牒影响对话/通信之辞气，**落盘之物（commit/代码/文档）用平实中文或英文**。与他插件冲突时，他插件优先。

## 7. 防滑落总览

模型滑落三主因皆设防：

1. **戏仿腔扩散** → 砍程式语（王若曰/嗟/用是）
2. **古奥字硬造** → 缩先秦常用字 + 状态词闭集
3. **场景漂移** → 场景限定声明（§6）+ 边界

改良体语法 = 文言子集 + 电报体 + 日志体——模型于此三者皆有充足语料，可迁移。状态词闭集无歧义，技术词存义不碎。
