# 牒

**牒（die）** 者，改良尚书体通信之插件也。启之，Claude Code 即以**改良尚书体**与子相答——古为密度之用，省 token 而人可读、模型不滑落。**电报之古简，通信之密语。**

姊妹插件 [`wen`](https://github.com/F2077/wen)（文）：文者对话之古辞，服务于审美；牒者通信之古简，服务于密度。一文一牒，一语一记，一散一密。

## 置

```
/plugin marketplace add https://github.com/F2077/die
/plugin install die@die-marketplace
```

置毕即行，毋需他设。欲止之：

```
/plugin disable die
```

## 理

以 `SessionStart` hook 于每会话之初注入牒之文法至上下文。**唯改辞气，不涉工程**——commit、注释、文档诸落盘之物，咸用平实中文或英文，不为牒所染。

牒以尚书之骨（省主语、一事一记、状态前置、助字分工）为根，砍其戏仿腔（王若曰/嗟/用是）与古奥字，缩至先秦常用字，设状态词闭集。核心为**条件省**——主语可推则省，不可推则补——故工作流极密，日常对话遇歧自补。详参本仓 [`牒法要略.md`](references/牒法要略.md)。

## 密

同一意，白话与牒之别（CI/CD 状态示例）：

| | 字数 | 折 token（约）|
|---|---|---|
| 白话「代码已经拉下来了。依赖装好了，没有冲突。构建跑起来了，还在进行。单元测试通过了，43 个全过。集成测试失败了，超时，可能是数据库连接池太小。正在查。部署没开始，等审批。」 | ~80 | ~55 |
| 牒「拉码毕。依赖装，无虞。构建起，行中。单测尽过，计四十有三。集测败，超时，疑连池隘。方察。部署待，候决。」 | ~38 | ~26 |

牒之令约 4,300 字符，折 token 约 1,100，**每会话一注**，后不复加。

> **电报之古简，通信之密语。**
